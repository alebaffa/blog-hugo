Be careful with slice

Posted on March 31, 2016 in golang, slices

Slices are a great tool in Go, but sometimes they can be confused with arrays. A small reminder: slices are a pointer into an underline array. Let’s take this example: array := []int{1, 2, 3, 4, 5, 6} slice := array[1:3] // let's print both fmt.Println("array: ", array) fmt.Println("slice: ", slice) // array: [1 2 3 4 5 6] // slice: [2 3] Since the slice is only a pointer to the underline array, if you do some operations with the elements of the array you are also changing the elements of the slice.
